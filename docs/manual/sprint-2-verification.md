# Sprint 2 — Develop マニュアル動作確認チェックリスト

実装完了後、Android / iOS の双方で以下を確認する。

## 事前準備

- 当日撮影済みの写真があると未現像泡が表示される。
- 24時間以上前の撮影があれば現像済み泡が表示される。
- 既存写真が無い場合は端末の日時を進めて検証するか、Sprint 1 のカメラ機能で複数回撮影しておく。

## 基本ナビゲーション

- [ ] アプリ起動 → HomeScreen が表示される
- [ ] HomeScreen 下部に「泡たち」（左下）「メニュー」（右下）ボタンが見える
- [ ] 「泡たち」タップ → GalleryScreen に遷移
- [ ] GalleryScreen の「ホーム」 → HomeScreen に戻る

## ギャラリー表示

- [ ] 写真が無いとき「まだ泡がありません」と表示
- [ ] 撮影直後の写真は未現像泡として表示（透明＋「あと○h」）
- [ ] 24時間経過後の写真は現像済み泡（写真が中に見える）
- [ ] 泡が縦スクロールで並ぶ
- [ ] 泡がそれぞれふわふわ揺れる（位置と微スケールアニメーション）
- [ ] 最新が一番上、古いものが下
- [ ] 泡のサイズと水平オフセットが写真ごとに毎回同じ（再起動しても同位置）

## 詳細画面

- [ ] 現像済み泡タップ → PhotoDetailScreen 遷移（fade + scale トランジション）
- [ ] 未現像泡タップ → 反応なし
- [ ] 写真がフルスクリーン近くで表示される
- [ ] 撮影日時とエリアラベルが表示される
- [ ] メモが表示される（空欄は「メモなし」）
- [ ] ✎ アイコンまたはメモテキストタップで編集モード
- [ ] 100文字を超えてキー入力しても 100文字でカット
- [ ] 「保存」タップ → 即座にメモが反映される（楽観的更新）
- [ ] 「キャンセル」タップ → 編集前のメモに戻る

- [ ] 左右スワイプで前後の現像済み写真に移動

## テーマ

- [ ] パレット選択で背景が変わる（HomeScreen / GalleryScreen / PhotoDetailScreen 全画面で反映）

## 既知の制限事項（Sprint 4+ で対応）

- 泡割れアニメーション（タップ時の 800ms バースト演出）は仕様 §3.6 に記載のとおり Sprint 4+ に延期。Sprint 2 では汎用の fade + scale トランジションで遷移する。
- 泡の `border-radius` モーフィングによる有機的な変形（CSS のシャボン玉風揺らぎ）も Custom Shape 実装が必要なため Sprint 4+。
- 泡の個別色変更（写真ごとのお気に入り色設定）は Sprint 4+。
- シェア機能は Sprint 3 で実装予定（ボタン位置のみ確保）。

## アクセシビリティ留意事項（後続スプリントで対応）

- BubbleGalleryItem の `AsyncImage` に `contentDescription` 未設定（Sprint 4+ a11y パスで対応）
- BottomActionBar / TextButton の `Role.Button` semantics 明示なし（同上）
